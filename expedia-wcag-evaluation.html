Evaluation Report

1. Executive summary

This report describes the conformance of the Expedia website with W3C’s Web Content Accessibility Guidelines (WCAG). The review process is described in Section 5 below and is based on evaluation described in Accessibility Evaluation Resources.

Based on this evaluation, the Expedia.com website does not meet WCAG 2.1, Conformance Level A. Detailed review results are available in sections below. Feedback on this evaluation is welcome.

2. Background about the evaluation

Conformance evaluation of web accessibility requires a combination of semi-automated evaluation tools and manual evaluation by an experienced reviewer. The evaluation results in this report are based on the evaluation conducted on the following date: 20 Jan 2018. The website may have changed since that time.

5. Review process

Identify WCAG 2.1 Level for which conformance was tested: WCAG 2.1 Level A and Level AA

Identify evaluation and validation tools used, and versions thereof: WAVE evaluation tool for Chrome version1.0.9

Description of manual reviews (usability testing of accessibility features) used:

On doing the manual tests I observed the heading level structural, content errors and discovered the different errors for the buttons, alt-text of images etc. on all the Pages listed above for which I decided to do an evaluation study. I have listed two errors from the Expedia homepage such as:

  1. The Home page has around 15 images with missing the alternative text.
  2. There are 19 instances where there are empty links, i.e., a link contains no text.

6. Results and recommended actions

Content-based issues

  1. Missing alternative text

  2. Missing form label

    3. Solution: If a text label for a form control is visible, use the <label> element to associate it with its respective form control. If there is no visible label, either provide an associated label, add a descriptive title attribute to the form control, or reference the label(s) using aria-labeled. Labels are not required for an image, submit, reset, button, or hidden form controls.

  3. Images used as a link without having a description or an alt text

    4. Solution: Provide appropriate alternative text and link description such as, “Image of Central Park, New York from 2008. Click to view the attraction.”

Color contrast issue - This is a design based issue

Problem: The contrast for the user details is not compliant with the WCAG guidelines.

Solution: Change the color codes to make the contrast compliant

Before: Color: #EDEDED

After: Color: #474747

Interaction based issue

Problem: On hover, there is an additional text that is shown. Along with this is is a clickable element which takes the users to a different page. It is highly likely that while using accessible technologies users will end up wrongly interacting with the system.

Solution: Show the text upfront instead of like a hover interaction. Multiple interactions on an element end up making the element difficult to use and are susceptible to mistaken clicks.
